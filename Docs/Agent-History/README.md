# Agent Implementation History

This directory stores task-level implementation history generated by agents.

## Purpose

Each task should leave a concise, auditable Markdown record that explains what changed, how it was verified, and where the implementation can evolve next.

## Required File Rule

- Create exactly one history file per implementation task.
- Place files in this directory.
- Use the filename format: `YYYY-MM-DD_HHMM_<task-slug>.md`
- Use local time in 24-hour format.
- Keep `<task-slug>` ASCII and lowercase kebab-case.

Examples:

- `2026-04-09_1430_add-self-update-validation.md`
- `2026-04-10_0915_fix-project-index-lookup.md`

## Required Sections (in order)

1. Added/Changed Files
2. Implementation Summary
3. Key Functions
4. Verification Steps
5. Future Extension Points

Use `TEMPLATE.md` as the canonical format.
